-----------Работа с конструкторами----------
1. Конструкторы нужны, чтобы определять способы создания экземпляров класса
и указывать значения полей экземпляров класса уже при создании экземпляра
Если не создан конструтор при описании класса, при создании экземпляра класса автоматически
создаётся конструктор по умолчанию и всем полям экземпляра класса присвамиваются значения
по умолчанию (в зависимости от типа поля)
public class Building {
    private String material;
    private String earCreate;
    private String typeFoundation;

    public Building() { //инициализация конструктора без параметров
        }
}

2. Объявление конструкторов с параметрами: причём, если определён конструктор с параметрами, то без прааметров уже
возможности вызвать не будет, придётся его определять самостоятельно в теле класса.
public Building(String material, String earCreate, String typeFoundation) {
        this.material = material;
        this.earCreate = earCreate;
        this.typeFoundation = typeFoundation;
    }

3. При создании  экземпляра класса-наследника сначала вызывается конструтор родительского класса, а затем уже
конструктор класса наследника
